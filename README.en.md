# AgentDeck

[中文 README](./README.md)

AgentDeck is a single-file HTML presentation toolchain for people and AI agents.

Its product promise is simple:

**Wrap presentation content from any source into one shareable, playable, exportable enhanced HTML file.**

## Product Boundary

AgentDeck is not trying to replace PPT skills, visual template authors, or presentation design systems.

It focuses on a lower-level infrastructure layer:

- Add a unified browser presentation player to an existing deck
- Wrap HTML decks generated by external PPT skills into a single HTML file
- Provide a basic Markdown path when no external skill is involved
- Provide presentation controls such as navigation, overview, next-slide preview, autoplay, blank screen, spotlight, fullscreen, and print / PDF
- Provide export and compatibility checks so agent-produced decks are easier to deliver

External PPT skills own style, aesthetics, templates, and content generation. AgentDeck owns packaging, playback, export, and compatibility.

## Third-Party Skills And Attribution

AgentDeck can work with PPT skills written by different authors, including open-source visual skills, futuristic deck skills, internal company template skills, or hand-written HTML decks.

Those skills are not part of AgentDeck. When using one, agents should clearly tell users:

- who authored the skill
- which license applies
- whether commercial use, modification, or redistribution is allowed
- that AgentDeck only downloads, calls, checks, imports, or wraps that output
- that the third-party visual style, template design, and generation logic should not be described as AgentDeck's own work

This boundary matters. AgentDeck should be infrastructure for the PPT skill ecosystem, not a project that absorbs other people's design work.

## Two Entry Points

### 1. Wrap an Existing HTML Deck

Use this when a deck has already been produced by a PPT skill, a web slides tool, hand-written HTML, or another agent.

```bash
agentdeck wrap-html path/to/index.html --out dist
open dist/index.html
```

`wrap-html` tries to detect common slide containers such as `.slide`, `.page`, `.ppt-slide`, `.swiper-slide`, and `section`. If no clear slide containers are found, it wraps the entire `<body>` as one slide.

### 2. Build A Basic Deck From Markdown

Use this when there is no external PPT skill and you want a lightweight source file.

```bash
agentdeck init my-deck --theme swiss
agentdeck build my-deck/deck.md --single-html --mode audience --out my-deck/dist
open my-deck/dist/index.html
```

This path is a basic authoring option, not the whole product positioning.

## CLI

```bash
agentdeck wrap-html path/to/index.html --out dist
agentdeck init my-deck --theme swiss
agentdeck lint my-deck/deck.md
agentdeck build my-deck/deck.md --single-html --mode audience --out my-deck/dist
agentdeck export my-deck/deck.md --pdf --png --long-image --grid9 --social-pack --out my-deck/export
agentdeck compat swiss-locked path/to/index.html
agentdeck import-swiss-locked path/to/index.html --out deck.md
```

`classify` and `adapt` still exist, but they are optional structure helpers rather than the main product idea:

```bash
agentdeck classify my-deck/deck.md
agentdeck adapt my-deck/deck.md --scenario pitch --out my-deck/deck.pitch.md
```

## What The Single HTML Includes

The generated or wrapped audience HTML includes built-in presentation controls:

- previous / next
- restart
- autoplay
- autoplay interval switch
- autoplay looping
- seek bar
- overview thumbnails with click-to-jump
- small next-slide preview in the bottom-right corner
- blank screen
- spotlight
- fullscreen
- browser print / PDF

Useful shortcuts:

- `ArrowLeft` / `ArrowRight`
- `O` overview
- `C` next-slide preview
- `B` blank screen
- `L` spotlight
- `P` autoplay
- `F` fullscreen
- `Esc` close active overlays

## How Agents Should Use It

Treat AgentDeck as the final-mile delivery tool:

1. The user selects or names a PPT skill, or chooses not to use one.
2. The agent clearly states the source, author, and license boundary of that skill.
3. The agent creates a deck using the external skill, hand-written HTML, or Markdown.
4. If HTML already exists, run `agentdeck wrap-html path/to/index.html --out dist`.
5. If the source is `deck.md`, run `agentdeck lint` and `agentdeck build`.
6. Open `dist/index.html` and check playback, overview, next-slide preview, blank screen, spotlight, and PDF.
7. If a third-party deck has compatibility risks, report those risks without presenting the third-party template as AgentDeck's own capability.

Agent rules:

- credit third-party authors
- do not copy unauthorized templates
- do not describe external skill aesthetics as AgentDeck originals
- prefer wrapping external outputs instead of rewriting their visual systems
- when asked to download or use a skill, state its source and license first

## Compatibility

Compatibility support uses neutral profiles:

- generic external HTML: `external-html`
- locked Swiss-style HTML: `swiss-locked`
- package: `@agentdeck/compat-profiles`

Commands:

```bash
agentdeck wrap-html path/to/index.html --out dist
agentdeck compat swiss-locked path/to/index.html
agentdeck import-swiss-locked path/to/index.html --out deck.md
```

`swiss-locked` is a structural compatibility profile for validating and importing locked HTML decks. It is not an AgentDeck brand and does not imply ownership of any third-party skill's template design.

## Minimal `deck.md`

```md
---
title: AgentDeck Single HTML Presentation
subtitle: Add a shareable HTML player to decks from any source
theme: swiss
outputs: [html, pdf, png]
mode: audience
---

# AgentDeck Single HTML Presentation
layout: cover

Third-party skills own content and visuals. AgentDeck owns the single-file HTML player.

# Product Boundary
layout: statement

AgentDeck does not replace PPT skills or presentation designers. It turns decks into playable, shareable, exportable single HTML files.
```

## Repository Layout

- `packages/runtime`: single-file HTML player and static renderer
- `packages/cli`: the `agentdeck` command
- `packages/compat-profiles`: external HTML and compatibility profiles
- `packages/schema`: Markdown DSL, types, and validation
- `packages/themes`: base themes and layout manifests
- `packages/skill`: agent-facing workflow guidance
- `examples/`: public examples
