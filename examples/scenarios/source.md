# AI 内容生产协作平台
layout: cover

团队已经有很多 AI 工具，但缺少一个能把策略、内容、视觉和交付物统一管理的生产协议。

# 现在的问题
layout: statement

内容团队不是缺少生成能力，而是缺少可复盘、可审查、可交付的协作流程。

# 产品方案
layout: cards

- 用 Markdown DSL 作为唯一内容源。
- 用场景模板把同一份内容重排成不同交付形态。
- 用视觉检查避免控件遮挡、文本溢出和图片错位。
- 用单 HTML 把播放、导出和传播统一到一个文件。

# 用户与市场
layout: table

| 用户 | 场景 | 价值 |
| --- | --- | --- |
| 自媒体作者 | 长文拆卡、九宫格、公众号头图 | 提高传播效率 |
| 创业团队 | 路演、融资、产品发布 | 快速讲清商业判断 |
| 教师讲师 | 课程、讲义、互动题 | 降低备课成本 |
| 供应商团队 | 讲标、评分点响应、案例证明 | 提升方案可信度 |

# 增长路径
layout: timeline

- 先服务高频内容创作者，形成社媒传播样例。
- 再进入创业路演和发布会场景，补齐商业模板。
- 最后沉淀讲标、教学和团队品牌规范。

# 风险与约束
layout: checklist

- 如果只做模板，会被已有 slides 工具替代。
- 如果只做 Agent prompt，无法稳定导出。
- 如果不做场景判断，用户不知道该选什么结构。
- 如果不做视觉 QA，单 HTML 也会变成不可用截图。
